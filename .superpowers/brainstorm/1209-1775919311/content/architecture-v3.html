Agent Memory Model (MS Agent Framework + Redis)

Memory Layers
HistoryProvider
• Chat messages per session
• before_run: load last N messages
• after_run: persist new messages
→ Redis Hash: agent:history:{session_id}
TTL: 24h (active sessions)
→ PostgreSQL: agent_sessions (archive)
ContextProvider
• User preferences & facts
• before_run: inject user context
• after_run: extract & store facts
→ Redis JSON: agent:ctx:{user_id}
Persistent (no TTL)
→ PostgreSQL: agent_memory (backup)
SemanticMemory
• Embeddings via Voyage-3
• before_run: find similar past context
• after_run: embed & store conversation
→ Redis Vector: agent:vec:{user_id}
HNSW index, cosine similarity
Semantic Cache (LangCache pattern)
Domanda utente → embed → search similar cached Q&A → hit? return cached → miss? call Claude → cache response
Riduce costi API Anthropic per domande ricorrenti
